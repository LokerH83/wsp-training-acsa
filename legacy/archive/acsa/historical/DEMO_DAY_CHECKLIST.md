# Demo Day Checklist

## Before The Meeting

- [ ] Open the static demo locally.
- [ ] Confirm Overview / Dashboard loads.
- [ ] Confirm Load Workbook opens.
- [ ] Click Load Sample Workbook.
- [ ] Confirm Manage Training opens.
- [ ] Search provider/course.
- [ ] Confirm Book Training is visible.
- [ ] Confirm People Profiles opens.
- [ ] Confirm Reports opens.
- [ ] Confirm Copy Executive Summary works.
- [ ] Confirm Reset Demo restores values.

## Power Apps Check

- [ ] Refresh Power Apps Studio / live app.
- [ ] Open Employee Profiles.
- [ ] Confirm view is `Client Demo - Employee Profiles`.
- [ ] Confirm `ID Number` is not visible.
- [ ] Confirm no private export button is shown.
- [ ] Keep Power Apps ready only as the future-state / Tier 3 proof.

## Files To Have Open

- [ ] `START_HERE_FOR_DEMO.md`
- [ ] `docs/ACSA_DEMO_SCRIPT.md`
- [ ] `docs/ACSA_THREE_TIER_PROPOSAL.md`
- [ ] `docs/ACSA_TOMORROW_MEETING_PACK.md`
- [ ] `exports/README.md`

## Meeting Flow

- [ ] Start with the business problem.
- [ ] State that the public/static demo uses synthetic data only.
- [ ] Show the dashboard first.
- [ ] Show workbook intake.
- [ ] Show planning and booking.
- [ ] Show reports.
- [ ] Show Power Apps as the governed app direction.
- [ ] Recommend Tier 2: Excel + Power BI Reporting.
- [ ] Close on discovery and data review as the next step.

## Do Not Show

- [ ] Raw private data.
- [ ] ID Number fields.
- [ ] Private snapshot folders.
- [ ] Old build attempts.
- [ ] VS Code chat or internal notes.
- [ ] Dataverse tables unless the safe view is confirmed.

## Fallback

- [ ] If GitHub Pages fails, run the local static demo.
- [ ] If Power Apps is slow, skip it and use the static demo plus proposal.
- [ ] If browser sharing is risky, use the one-page summary and demo script.
