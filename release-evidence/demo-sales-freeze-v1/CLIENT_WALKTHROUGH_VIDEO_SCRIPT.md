# Client Walkthrough Video Script

Purpose: record a 3-5 minute sales walkthrough of the static WSP / ATR reporting demo.

Recommended title:

SkillSet SA Training Reporting Hub — WSP / ATR Demo Walkthrough

Recording setup:

- Open the live demo: https://lokerh83.github.io/wsp-training-acsa/
- Use a clean browser window.
- Zoom between 90% and 100%.
- Keep the recording calm and practical. This should feel like a business walkthrough, not a product advert.
- Do not upload or show private client data. Use the synthetic demo data only.

## 1. Opening: the management problem

Suggested narration:

Many training and SDF teams still rely on workbook-based processes to plan training, capture bookings, track completion and prepare WSP or ATR reporting. The problem is not usually that Excel is bad. The problem is that the reporting process becomes difficult to control once data sits across multiple files, manual updates, repeated checks and separate evidence trails.

This demo shows how that process can be turned into a more controlled reporting workflow.

Show on screen:

- Overview page
- Hero area
- Key dashboard cards

## 2. How the demo works

Suggested narration:

The demo uses synthetic data only. It is designed to show the workflow rather than act as a production system. A real client version would run inside the organisation's Microsoft 365 environment with controlled access, central data storage and proper audit history.

The flow starts with training data, employee records, providers, courses, planned training and achieved training.

Show on screen:

- Load Workbook
- Demo data controls
- Data Quality Centre

## 3. Booking and ATR flow

Suggested narration:

From here, the team can select an employee, view profile context, choose a provider or course and capture a booking. Completed training can then be recorded against ATR activity with evidence context.

The important part is that the booking and reporting process becomes visible. Instead of relying on one person remembering which workbook was updated, the process creates a structured trail.

Show on screen:

- Training Management
- Employee selector
- Provider/course selection
- Booking form
- ATR action area

## 4. Key reporting views

Suggested narration:

The reporting view brings the operational activity back into management language. It shows requested, planned and achieved training, highlights gaps and makes the reporting cycle easier to explain.

For a production client, this is where Power BI can be connected to governed data so management can see progress by division, department, region, role, demographic category or reporting period.

Show on screen:

- Reports
- Filters
- KPI cards
- Requested versus planned versus achieved view
- Export button

## 5. Business value

Suggested narration:

The value is not just a nicer screen. The value is control. The organisation can reduce manual consolidation, improve evidence readiness, see reporting gaps earlier and create a repeatable process that does not depend on one fragile workbook or one person holding all the knowledge.

Show on screen:

- Submission Workspace
- Action register
- Readiness indicators

## 6. What production would add

Suggested narration:

This static demo is the sales version. The production pilot would add a Microsoft 365 backend: SharePoint or Dataverse storage, user permissions, evidence attachments, approval workflow, audit history, duplicate prevention, controlled imports and Power BI reporting.

That is the next step after a client wants to pilot the system.

Show on screen:

- Overview again
- End on the hero/dashboard

## Closing line

Suggested narration:

This is the difference between keeping training reporting as a collection of files and turning it into a managed reporting system.

## Quick checklist before sending the video

- The video is between 3 and 5 minutes.
- No private workbook or client data is visible.
- The demo URL is visible only if you want the viewer to test it.
- The production boundary is clear: static demo now, Microsoft 365 pilot next.
- The final CTA is a pilot discussion, not a generic software sale.
