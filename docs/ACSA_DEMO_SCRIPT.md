# ACSA SDF Demo Script

Target time: 5 to 7 minutes.

## 1. Opening

Explain:

- This is a demo-safe prototype.
- It uses synthetic data.
- It is designed to show the SDF workflow concept.
- It is not production and not official SETA/B-BBEE output.

Suggested wording:

“Today I want to show a practical concept for how ACSA could manage WSP and ATR reporting more clearly. This demo is not production data. It is a demo-safe version showing the workflow from workbook loading through to training planning, booking, people profiles and reporting.”

## 2. Overview

Show:

- Requested / Suggested
- Planned WSP
- Achieved ATR
- Gaps
- Review Items and reporting gaps
- Review items

Key message:

“This gives the SDF and management team a quick view of where the training plan is complete, where activity has happened, and where there are gaps.”

## 3. Load Workbook

Show:

- Load Sample Workbook
- Staging
- Required fields
- Import preview

Key message:

“This is where Excel remains useful, but becomes controlled. The production version would validate the data before it is accepted into the reporting or system layer.”

## 4. Manage Training

Show:

- Add requested training.
- Add planned WSP item.
- Search provider/course.
- Use selected course.
- Book training.
- Record ATR.

Key message:

“This connects the planning process to the training action. It moves away from disconnected spreadsheets.”

## 5. People Profiles

Show:

- Search a person.
- Requested training.
- Planned training.
- Booked training.
- Achieved training.
- Demographics.

Key message:

“The employee profile becomes the link between the workbook, WSP planning, ATR achievement and reporting.”

## 6. Reports

Show:

- Filter by provider.
- Filter by course.
- Filter by region/cluster.
- Filter by race.
- Filter by sex/gender.
- Filter by age band.
- Filter by disability.
- Confirm the row count and active filters.
- Export the filtered report as an Excel-ready CSV.
- Open the exported CSV in Excel.
- Or copy the filtered table and paste it into Excel.
- Copy executive summary.

Key message:

“This is the management view. It supports quicker review of requested vs planned vs achieved training, and it gives the SDF user a filtered extract for Excel or email.”

## 7. Close

Explain the three options:

- Tier 1: Excel Control Pack
- Tier 2: Excel + Power BI Reporting
- Tier 3: Full Microsoft App Build

Recommended close:

“My recommendation is to start with the Excel and Power BI reporting layer first. That proves the data model and reporting logic before ACSA invests in the full app build.”
