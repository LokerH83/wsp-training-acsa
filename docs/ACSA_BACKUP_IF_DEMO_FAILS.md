# Backup Plan If Demo Fails

## What To Say

“This is a demo-safe prototype, so the important part is the workflow rather than the live technology. I can still walk through the concept using the proposal structure.”

## Fallback Flow

1. Explain the problem.
2. Explain the workbook-to-reporting workflow.
3. Explain the three proposal tiers.
4. Recommend Tier 2.
5. Offer the next step: discovery and data review.

## Key Message

The safest first step is to prove the reporting layer before investing in full workflow automation.

## Do Not Overpromise

Avoid saying:

- this is production-ready
- this is official SETA output
- this is official B-BBEE output
- this replaces all ACSA systems immediately

## Safe Wording

“This is a practical route to move from disconnected spreadsheets to cleaner reporting and, later, a governed Microsoft app if required.”
