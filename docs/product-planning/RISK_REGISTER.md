# Risk Register

| Risk | Impact | Likelihood | Mitigation | Status |
|---|---|---|---|---|
| Demo mistaken for production | High | Medium | Keep demo data labels visible and document limitations | [Partial] |
| Client expects official SETA automation | High | Medium | State that outputs support review, not official submission | [Partial] |
| Workbook structures differ by client | High | High | Run discovery and field mapping before pricing final delivery | [Needs Review] |
| WSP to ATR matching is imperfect | High | Medium | Use Employee Number, Course ID, Provider ID, Plan ID and date/quarter | [Needs Review] |
| Data quality is poor | High | High | Add import validation and review item workflow | [Needs Review] |
| User cannot maintain system | Medium | Medium | Provide training, handover and support options | [Needs Review] |
| Scope becomes too large | High | Medium | Use tiered roadmap and change request process | [Needs Review] |
| Security not properly defined | High | Medium | Complete security/privacy plan before production | [Not Started] |
| Pricing underestimates complexity | High | Medium | Complete discovery and size by data complexity | [Needs Review] |
| Support effort is underestimated | Medium | Medium | Define support hours, exclusions and escalation route | [Needs Review] |
