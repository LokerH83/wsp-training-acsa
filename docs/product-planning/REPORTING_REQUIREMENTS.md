# Reporting Requirements

## Core Metrics

- Requested / suggested training count
- Planned WSP activity count
- Achieved ATR activity count
- Employees in scope
- Booked training
- Upcoming sessions
- Completed bookings
- Attendance to confirm

## WSP / ATR Gap Metrics

- Requested not planned
- Planned not achieved
- Achieved not planned
- Review items
- Reporting gaps
- Completion percentage

## Budget / Spend Metrics

- Annual payroll estimate
- Estimated SDL
- Potential mandatory grant estimate
- Grant value at risk estimate
- Evidence / review gap count
- B-BBEE skills risk signal

All financial values are demo planning estimates only and must be confirmed through the client's approved process.

## Required Filters

- Date range if available
- Month if available
- Quarter if available
- Year if available
- Region / Cluster
- Division
- Department
- Provider
- Course / Intervention
- Sex / Gender
- Race
- Age Band
- Disability
- Status
- Booking Status
- Evidence Status if available

## Report Coverage Requirement

The report must show:

- Earliest date
- Latest date
- Included months
- Missing months
- Quarter coverage
- Year coverage
- Row count
- Workbook name
- Load date

If exact dates are unavailable, the report must show available quarter/period coverage and say that exact date coverage is not available from the current fields.

## Export Requirement

The export must include:

- Report title
- Date range covered
- Active filters
- Export date
- Filtered row count
- Management matrix rows

For the current static demo, the exported CSV contains the filtered management matrix rows. Report title, coverage and active filter context are visible in the UI and should be added as metadata rows in a future production export if the client requires it.
