# Pilot Run Plan

## Purpose

Run one controlled pilot with client-safe workbook data to prove the reporting workflow before committing to a larger build.

## Pilot Scope

- Load a client-safe workbook sample.
- Confirm field mapping.
- Confirm report coverage.
- Filter the Management Matrix.
- Export filtered CSV.
- Copy filtered table.
- Review key gaps with the SDF.

## Inputs Needed

- [ ] Client-safe sample workbook.
- [ ] Field descriptions.
- [ ] Reporting period.
- [ ] Provider/course list.
- [ ] Pilot user names.
- [ ] Success criteria.

## Pilot Steps

1. Review workbook structure.
2. Map fields to the data dictionary.
3. Load sample data into a controlled demo/test environment.
4. Check report coverage.
5. Apply management filters.
6. Export filtered report.
7. Compare output with the client's expected view.
8. Record gaps and changes.
9. Decide whether Tier 2 or Tier 3 is the next step.

## Success Criteria

- [ ] SDF can explain what the report covers.
- [ ] Filters produce useful management views.
- [ ] Exported rows match visible filtered rows.
- [ ] Gaps are understandable.
- [ ] Next commercial step is clear.

## Pilot Output

- Pilot findings summary
- Data quality issues list
- Required workbook changes
- Reporting design changes
- Commercial recommendation
