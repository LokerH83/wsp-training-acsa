# ACSA Tier 2 Discovery Checklist

## Purpose

Use this checklist to guide the Tier 2 discovery and data review session for ACSA. The goal is to confirm the current Excel workbook structure, data requirements, reporting expectations, and the next steps for the Excel + Power BI delivery.

## Session Objectives

- Understand the current workbook and data sources
- Confirm required fields for employees, providers/courses, WSP and ATR
- Identify evidence and review item requirements
- Define management reporting questions and dashboard needs
- Agree the Tier 2 implementation scope and next actions

## Participants

- SDF / HR representative
- Training / ATR owner
- Reporting stakeholder or manager
- IT / data support (if available)
- Consultant / delivery lead

## Checklist

### 1. Workbook & Data Structure

- [ ] Review existing workbook tabs and purpose
- [ ] Confirm the current employee master table fields
- [ ] Confirm the WSP planning table fields
- [ ] Confirm the ATR actuals table fields
- [ ] Confirm provider/course catalogue fields
- [ ] Confirm evidence register fields
- [ ] Confirm review items table fields
- [ ] Identify any additional workbook tables in use
- [ ] Confirm if workbook is already using named tables
- [ ] Capture any workbook file naming or versioning rules

### 2. Employee & Demographic Data

- [ ] Confirm employee identifiers used in the workbook
- [ ] Confirm mandatory employee fields for reporting
- [ ] Confirm available demographics:
  - Cluster / Region
  - Division / Department
  - Race
  - Sex
  - Age or age band
  - Disability status
- [ ] Confirm whether any additional employee attributes are required

### 3. Training Planning & ATR Rules

- [ ] Confirm how requested training is recorded
- [ ] Confirm how planned training is recorded
- [ ] Confirm how actual training (ATR) is recorded
- [ ] Confirm training statuses and completion indicators
- [ ] Confirm provider/course linkage rules
- [ ] Confirm booking or attendance data requirements
- [ ] Confirm any business rules for requested vs planned vs achieved logic

### 4. Evidence & Review Items

- [ ] Confirm what evidence types are tracked
- [ ] Confirm evidence readiness categories/statuses
- [ ] Confirm how evidence is linked to training records
- [ ] Confirm review item categories and owners
- [ ] Confirm required review item status tracking
- [ ] Confirm any escalation or follow-up reporting needs

### 5. Reporting Requirements

- [ ] Confirm the key questions management needs answered
- [ ] Confirm the priority dashboard pages or sections:
  - Executive summary
  - Requested vs Planned vs Achieved
  - Provider / course analysis
  - Evidence readiness
  - Review items
- [ ] Confirm required drill-down filters:
  - Cluster / Region
  - Division / Department
  - Race / Sex / Age / Disability
  - Provider / Course
- [ ] Confirm any additional filter or slicer requirements
- [ ] Confirm preferred refresh frequency and update process
- [ ] Confirm who will consume the Power BI report

### 6. Deployment & Handover

- [ ] Confirm who owns the workbook maintenance
- [ ] Confirm how workbook updates will be published to Power BI
- [ ] Confirm training needs for SDF and managers
- [ ] Confirm handover expectations and documentation needs
- [ ] Confirm next milestone for Tier 2 delivery

## Recommended Outputs

- Data inventory of the workbook fields and tables
- Agreed list of report pages and key metrics
- Agreed list of filters and supporting dimensions
- Handover notes for the workbook-to-Power BI refresh process
- Next step plan for building Tier 2

## Notes

This checklist supports the Tier 2 delivery approach in `docs/ACSA_TIER2_IMPLEMENTATION_PLAN.md` and should be used before design and build begin.
